# Resume Generator - Electron Application

This project has been packaged as a standalone Windows Electron application. Users can now run it as a native app without any command-line knowledge.

## Quick Start for End Users

### Simplest Method: Click `Launch-App.ps1`

1. Right-click **`Launch-App.ps1`** in the project folder
2. Select **"Run with PowerShell"**
3. The app will launch automatically (dependencies installed on first run)

Or, if that doesn't work:
1. Open PowerShell
2. Navigate to this folder
3. Run: `.\Launch-App.ps1`

### What Happens?

On first run:
- вң“ Checks that Node.js and Python are installed
- вң“ Installs Node.js dependencies
- вң“ Creates a Python virtual environment
- вң“ Installs Python dependencies
- вң“ Launches the Electron app

On subsequent runs:
- вң“ Skips setup (already done)
- вң“ App launches in ~5-10 seconds

## For Developers

### Development with Hot Reload

```bash

npm run electron:dev
```

This runs:
- Vite dev server on port 5174 (auto-reload on code changes)
- Python backend on port 8002
- Electron window pointing to the dev server

Both are ready to distribute to end users. They contain:

- One-click launch

## Troubleshooting

### Issue: "Node.js not found"
**Solution:** Install Node.js from https://nodejs.org/
- Download the LTS version
- Run the installer
- Restart PowerShell/terminal

### Issue: "Python not found"
**Solution:** Install Python from https://www.python.org/
- Download Python 3.11+
- **Important:** Check "Add Python to PATH" during installation
- Restart PowerShell/terminal

### Issue: Port 8002 already in use
**Symptom:** Backend fails to start

**Solution:** Kill the process using port 8002
```bash
# Find process using port 8002
netstat -ano | findstr :8002

# Kill it (replace PID with the actual PID)
taskkill /PID <PID> /F
```

### Issue: "Cannot find module" errors
**Solution:** Reinstall dependencies
```bash
cd web

# Clear cached dependencies
rm -r node_modules package-lock.json
npm install --legacy-peer-deps

# For Python issues
rm -r venv
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt
```

### Issue: Electron window won't open
**Check:** 
1. The backend is running (look for terminal output)
2. Port 8002 is accessible
3. Vite is compiling (in dev mode)

**Solution:** Try running manually:
```bash
cd web

# Install packages
npm install --legacy-peer-deps

# Terminal 1: Start backend
python -m uvicorn main:app --port 8002

# Terminal 2: Start frontend dev server
npm run dev:client

# Terminal 3: Start Electron
npx electron ..
```

## Configuration

### Data Folders

Users specify these in the app UI:
- **_generated**: Where global workbooks are stored
- **person_workbooks**: Where individual person workbooks are loaded

### Environment Variables

Create a `.env` file in the `web/` folder:
```env
ALLOW_EXTERNAL_PATHS=false
LOCAL_DATA_ROOT=../data
LOCAL_PURSUITS_ROOT=../outputs
```

These are read by the Python backend on startup.

## Performance Notes

- **First launch**: ~10-15 seconds (installing dependencies)
- **Subsequent launches**: ~5-10 seconds (startup only)
- **Build time**: ~30 seconds (production build)
- **Installer size**: ~500MB (includes Node.js, Python, all dependencies)
- **Portable executable size**: ~200MB

## Security

- Context isolation enabled (prevents XSS escaping)
- Preload script restricts renderer process capabilities
- Node integration disabled in renderer process
- All inter-process communication goes through Electron IPC (when implemented)
